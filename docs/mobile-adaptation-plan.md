# 移动端适配任务规划

JM Boom 当前主体验来自桌面端 Tauri 布局。Android 包可以运行，但移动端需要按手机漫画阅读 App 重新校准比例、触控区域、固定浮层和阅读器视口。

## 设计决策

- 配色：保留现有黑白中性色与阅读器深色背景，不引入新的品牌主色。
- 字体：继续使用 Geist Variable 与 Noto Sans SC Variable，只调整移动端字号层级。
- 间距：手机页面左右留白为 `16px`，平板为 `24px`，桌面保持原有 `32px` 节奏。
- 圆角：保留现有胶囊按钮和小圆角卡片，不新增装饰性嵌套卡片。
- 浮层：固定控件使用轻边框与 backdrop blur；底部导航允许轻阴影。
- 动效：保留 150-300ms 的透明度/位移动效，避免会造成布局跳动的 hover 动效。

## 断点与比例

- 手机：`360-430px`，重点按 `375x812` 校验。
- 平板：`768px`，内容网格以 3 列和不重叠筛选控件为目标。
- 桌面：`1024px+`，保留原有左侧浮动导航和宽屏布局。
- 漫画网格：手机 2 列，平板 3 列，桌面 4 列。
- 触控目标：主要控件至少 `44x44px`；阅读器密集控件可降到 `40x40px`，但需要保留间距。
- 安全区：所有固定底部控件都需要包含 `env(safe-area-inset-bottom)`。
